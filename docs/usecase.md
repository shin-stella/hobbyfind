# HobbyFind 유스케이스 문서 (Use Case Specification)

# 1. 액터 정의 (Actor Definitions)

| 액터                   | 설명                                   |
| -------------------- | ------------------------------------ |
| 비회원 사용자 (Guest User) | 로그인하지 않은 사용자. 취미 탐색 및 카테고리 조회 가능     |
| 회원 사용자 (Member User) | 로그인한 사용자. 취미 탐색, 북마크 관리, 마이페이지 이용 가능 |
| 시스템 (System)         | 인증, 데이터 조회, 북마크 저장 및 통계 제공 담당        |

---

# 2. 상세 Use Case 시나리오 (Detailed Use Case Scenarios)

## UC-01. 메인페이지 탐색

### 목적

사용자가 전체 취미 목록을 탐색한다.

### 액터

* 비회원 사용자
* 회원 사용자

### 사전조건

* 서비스 접속 가능 상태

### 사후조건

* 사용자가 취미 목록을 확인함

### 메인 흐름

| 단계 | 수행 내용            |
| -- | ---------------- |
| 1  | 사용자가 홈 페이지 접속    |
| 2  | 시스템이 전체 취미 목록 조회 |
| 3  | Hero 섹션 표시       |
| 4  | 취미 카드 그리드 표시     |
| 5  | 사용자가 취미 목록 탐색    |

---

### 사후조건

# 6. 비즈니스 규칙 및 제약사항 (Business Rules and Constraints)

## 사용자 규칙

* 회원가입 후 로그인 가능
* 비회원은 북마크 불가
* 비회원은 마이페이지 접근 불가

---

## 북마크 규칙

* 동일 취미 중복 저장 불가
* 회원별 북마크 데이터 독립 관리

---

## 취미 데이터 규칙

### 운동형

* 조깅/러닝
* 요가
* 수영
* 자전거
* 클라이밍
* 댄스

### 지능형

* 독서
* 퍼즐
* 체스
* 프로그래밍
* 외국어 학습
* 사진 촬영

### 예술형

* 그림 그리기
* 악기 연주
* 요리
* 서예
* 도자기 만들기
* 정원 가꾸기

※ 목록 추가·수정 불가
